"Level Types" (Candy Spawning RNG)

Used by level designers of King to modify candy spawning patterns depending on the style of level, for balancing.
Excavation: A board where you remove blockers in a simple, traditional manner.
Sniping: A board with elements that you can only access with special candies.
Puzzle: A complex board that usually involves very specific strategies to pass. Can also be used on the opppsite spectrum as a simple level with little to no blockers.
Endurance: A board that involves blockers that spread or return from cannons (toffee, chocolate, etc)
Limited Play Space: A board that starts off with very few open spaces, as a result of many sugar coats or other blockers. Can also be used for a level with only a few spaces.
Explosive: A level with low colors, or alternatively, a level with a lot of special candy spawns or chain reactions.
Add this code to add a spawning pattern (add or remove depending on the level styles that fit your level):

"levelType":["Excavation","Sniping","Puzzle","Endurance","LimitedPlaySpace","Explosive"],
Tile map codes How it works

Candy Crush is a 9x9 board game; therefore, every level is mapped in a 9x9 tile as well:

"tileMap":[
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"],
["000","000","000","000","000","000","000","000","000"]
]

Quite apparently, every "000" represents a tile, and every row is coded in a line (so there are 9 lines), in which every 3-character number constitutes an element. Here are some basic examples and notes:

  • "000" represents the empty space; "001" represents the empty tile, "002" represents a random-coloured candy; "044" represents a colour bomb.
  • The elements can also be stacked on the code, such that "002009" represents a random-coloured candy inside the chocolate; while "042044" represents a colour bomb inscribed a 5-layered lock.
  • Some blockers are not supposed to have anything such as candies/colour bombs inside the blocker, like icings. To ensure no candies are inside the blocker, a "006" is used to prevent bugs from happening. For example, "006023" represents a 5-layered icing, "006024" represents a chocolate spawner.
  • Candies spawn-point somehow should be available. Of course, it is not necessary if you want to make a level like clearing the whole board without candies left (which King removed this kind of gameplay by redesigning that only level). Examples:
    • "005" represents a spawn point in which every element, such as liquorice swirl, ingredients, and candies mentioned above on the code, is spawned (because it is not specified further, that will be an empty tile).
    • If you don't want everything to spawn at one single point, you need to specify what to spawn on that tile. For example, "002005026" means a random-coloured candy is included in the tile while a random-coloured candy spawn point is above it; "002005026027028" means a random-coloured candy is included in the tile while a random-coloured candy, ingredients and liquorice swirl are spawned at that point.
  • Remember:
    • By the game mechanics, blockers (of course, movable blockers) have the highest priority on spawning, then ingredients, and the last are candies. Therefore, if you set 20 liquorice swirls to be spawned every move, it is impossible for ingredients to be spawned at the whole game if you set them to be spawned at the same point.
    • It is possible to have an unmovable blocker and a spawn point exist together. For example, "005006023028" means the tile includes a 5-layered icing with an ingredient cannon above it. Therefore, don't make silly mistakes (unless you want to troll) such as "004005006024027" (ingredient cannon with a chocolate spawner and a double jelly underneath); no ingredients can be spawned at that point, and that jelly obviously cannot be cleared.
    • "00205" (instead of "002005") will make the game crash because it does not follow the 3-character number rule.
    • "002005 (instead of "002005") will also make the game crash because a close quotation mark is missing.
  • You may try out different combinations with creativity!

Teleporter

Teleporters, teleport the things from there to there. Remember, 14 is very important for teleporters' codes, it can hide, or show the elements in the board. Here is the use of teleporters:


Without 14 in the code

"portals":[[[3,4],[7,1]],[[8,3],[7,5]]],

With 14 in the code

"portals":[[[3,4,14],[7,1,14]],[[8,3,14],[7,5,14]]],

Sour Skulls

Using Sour Skulls's code may be complicated but not difficult.

Code:

"skulls":[[tile number,tile number,(1, 2, 3...),false/true,false/true]], (if any)
"skulls":[], (if none)

1, 2, 3,...: Sour Skull appear sequence.
First false/true: No starting point/starting point for the skull to appear on this pedestal.
Second false/true: The skull will/ will not appear on this pedestal.

Wonderful Wrapper

Possibly the longest code ever!

Code:

"wonderfulWrappers":[[From tile number, Orientation, Colour of 1st ribbon, Colour of 2nd ribbon, Colour of 3rd ribbon, Colour of 4th ribbon, Colour of 5th ribbon, 
1st element contained, 2nd element contained, Colour of 1st element, Colour of 2nd element]], (if any)
"wonderfulWrappers":[], (if none)
  • Orientation: 1 is vertical, 2 is horizontal.
  • Ribbon colours: -1 is no ribbon, 0 is blue, 1 is green, 2 is orange, 3 is purple, 4 is red, 5 is yellow.
  • Element colours: -1 is random coloured, 0 is blue, 1 is green, 2 is orange, 3 is purple, 4 is red, 5 is yellow.
  • Wonderful Wrapper elements contained: 44 is colour bomb, 45 is horizontal striped candy, 46 is vertical striped candy, 47 is wrapped candy, 49 is jelly fish.

Example:

"wonderfulWrappers":[[1,8,2,0,1,2,3,4,44,49,3,2],[2,0,1,3,3,-1,-1,3,49,47,5,3]],

How to use: Step-by-step

If you are a beginner in using Level Editor, check out these guides for making your first complete level.

  1. First, make a design for your level. Your level can be simple moves level, jelly level, ingredients level,... You can refer to some levels by candycrushgemer for making an idea for your level.
  2. Now, let's get started. Open Level Editor, now you are in the main editor.
    Starting with making your level, for example, if you want to place a red candy, first choose color in the Color dropdown menu, then make sure you have element selected in the Candy menu. Now, click on the grid to place the candy and you are done! Notes:
    • Mixers, Candy Cannon are elements that needed their own properties. You can change them in the Configuration side option.
    • You shouldn't add to many special candies otherwise the game will be unplayable due to the game mechanics.
    • Candy Cobra notes: if you don't make a level with Candy Cobra inside, when exporting you should delete the code line "candyCobras":3 at the end of the code. If you make a level with Candy Cobra inside, then remember to change
  3. Export your level: Click on File > Export level or you can use Ctrl + S keyboard shortcut to export your level. You can save your level to your device or share it to anywhere. Note if you want the link shorter please use the link shortener online cuz this editor is not supported short links yet.
  4. The next time you want to load your levels, please go to File > Open or use Ctrl + O keyboard shortcut to open files. You can also load levels using File > Import level or Ctrl + I. You can select your level or type in your level code.
